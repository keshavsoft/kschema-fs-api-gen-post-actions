ShowAll Action Documentation

Understanding how ShowAll works internally in @keshavsoft/kschema-api-gen-actions

1. Introduction

ShowAll is a CLI-generated Express.js GET endpoint. It automatically creates files, updates routes, and generates boilerplate code for fetching all records from a table.

Instead of manually writing controller, service, and route code, this npm package generates everything automatically using a modular and configuration-driven architecture.

2. Command Usage

npx @keshavsoft/kschema-api-gen-actions ShowAll

This command generates the ShowAll action inside the current endpoint folder.

4. How ShowAll Works

Step 1 - CLI Starts

The command is executed through cli.js. It loads the latest runtime version dynamically.

Step 2 - Command Resolution

actions.json acts as a registry. It maps the command name ShowAll to the corresponding action file. 

{
   "cmd": "ShowAll",
   "file": "showAll"
}

Step 3 - Template Generation

The generator locates the template folder and copies the required files into a new ShowAll folder. 

ShowAll/
 ├── controller.js
 ├── service.js
 ├── errors.js
 └── rest.http

Step 4 - Route Injection

The package automatically updates end-points.js.

It injects:

import { getFunc } from "./ShowAll/controller.js";

router.get('/ShowAll', (req, res) =>
    getFunc({
        res,
        inTableName : tableName
    })
);

Step 5 - API Ready

After generation, the endpoint becomes immediately usable. 

GET /ShowAll

5. Generated Files Explanation

controller.js

Handles HTTP request and response. Calls the service layer and manages error handling.

service.js

Contains business/data logic. Fetches all records using: 

kschema.table(tableName).query.findAll()

errors.js

Defines reusable custom error classes like:

  • ConflictError
  • StorageError

rest.http

Auto-generated API testing file for VS Code REST Client.

6. Internal Architecture

  • Config Driven - commands are registered in JSON
  • Dynamic Imports - actions load only when needed
  • Version Isolation - runtimes like v16, v17, v18 remain independent
  • Template Based - reusable templates generate code
  • Source Transformation - routes and imports update automatically

7. Why This Architecture?

This architecture helps developers:

  • Generate APIs quickly
  • Maintain clean project structure
  • Reduce boilerplate code
  • Scale endpoints easily
  • Keep controller/service separation clean

8. Conclusion

ShowAll is not just a CRUD generator. It demonstrates a modular CLI architecture using:

  • Dynamic runtime loading
  • Configuration-driven execution
  • Template-based scaffolding
  • Automatic source code transformation
  • Layered backend generation

The package is designed to help developers build scalable Express.js APIs with minimal manual setup.
